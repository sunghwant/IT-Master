07. 배열 (Array): 데이터 기차

수십, 수백 개의 변수를 하나하나 만들지 마세요. 기차처럼 하나로 연결해 드립니다!

1. 배열의 기초: 0번 칸부터 시작합니다!

변수가 상자 1개라면, 배열은 같은 크기의 상자들이 기차처럼 쭉 이어져 있는 형태입니다. C언어에서 가장 중요한 규칙은 상자 번호(인덱스)가 무조건 0번부터 시작한다는 것입니다.

▲ 배열의 구조: 5칸짜리 배열을 만들면 인덱스는 0부터 4까지 존재합니다.

💡 배열을 만드는 3가지 방법

// 1. 크기만 정해두고 나중에 값 넣기 (쓰레기값이 들어있음)
int arr1[5]; 

// 2. 선언과 동시에 값 채워 넣기 (가장 많이 씀)
int arr2[5] = {10, 20, 30, 40, 50};

// 3. 알아서 크기 맞춰달라고 하기 (자동으로 3칸짜리 배열이 됨)
int arr3[] = {1, 2, 3};

2. 🎒 기초 예제: RPG 아이템 인벤토리

배열의 진가는 for 반복문과 만났을 때 발휘됩니다. 배열의 인덱스(0, 1, 2...)를 반복문의 i로 대체하면 단 몇 줄의 코드로 모든 칸을 검사할 수 있죠!

#include <stdio.h>

int main() {
    int inventory[5] = {10, 20, 0, 5, 1};

    printf("--- 나의 인벤토리 ---\n");
    for(int i = 0; i < 5; i++) {
        printf("%d번 칸 아이템 수량: %d개\n", i, inventory[i]);
    }
    
    return 0;
}

3. 📊 심화 예제 1: 학생 점수 입력받고 평균 구하기

이번에는 반복문을 돌면서 scanf로 배열에 직접 데이터를 집어넣고, 다시 반복문으로 총합을 구해 평균을 내는 실용적인 프로그램을 만들어 봅시다.

#include <stdio.h>

int main() {
    int scores[3]; // 3명의 점수를 저장할 배열
    int sum = 0;

    // 1. 반복문으로 배열에 값 채워넣기
    for(int i = 0; i < 3; i++) {
        printf("%d번 학생의 점수를 입력하세요: ", i + 1);
        scanf("%d", &scores[i]); // & 기호 잊지 마세요!
    }

    // 2. 배열의 모든 값을 더하기
    for(int i = 0; i < 3; i++) {
        sum += scores[i];
    }

    printf("총점: %d점, 평균: %.2f점\n", sum, (float)sum / 3);
    
    return 0;
}

4. ⚔️ 심화 예제 2: 가장 강한 몬스터 찾기 (최댓값 알고리즘)

배열 안에 있는 데이터 중 가장 큰 값을 찾는 것은 프로그래밍의 아주 기본적이고 중요한 알고리즘입니다. 배열, 반복문, 조건문이 모두 결합된 완전체 예제입니다!

#include <stdio.h>

int main() {
    int monster_hp[5] = {450, 1200, 320, 980, 150};
    
    // 일단 0번 칸의 몬스터가 가장 세다고 가정합니다.
    int max_hp = monster_hp[0]; 
    int boss_index = 0;

    // 1번 칸부터 끝까지 싸움을 붙여봅니다.
    for(int i = 1; i < 5; i++) {
        // 내가 알던 최댓값보다 더 쎈 놈이 나타나면?
        if(monster_hp[i] > max_hp) {
            max_hp = monster_hp[i]; // 최댓값 갱신!
            boss_index = i;         // 보스의 위치 기록
        }
    }

    printf("던전의 보스는 %d번 칸에 있으며, 체력은 %d입니다!\n", boss_index, max_hp);
    
    return 0;
}

💀 초보자가 가장 많이 하는 실수: 선 넘지 마세요!

만약 int arr[5]; 라고 5칸짜리 배열을 만들었다면, 쓸 수 있는 인덱스는 0부터 4까지입니다. 실수로 arr[5] = 100; 처럼 5번 인덱스(6번째 칸)에 접근하려 하면, 컴퓨터가 다른 프로그램의 메모리를 건드렸다고 판단하여 에러를 내뿜거나 프로그램이 뻗어버립니다. 이를 배열의 범위를 벗어났다(Out of Bounds)고 합니다!
